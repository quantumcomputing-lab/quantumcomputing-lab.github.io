4sh.education — Site Structure Plan
=====================================
Last updated: 2026-06-25


AUDIENCE
--------
Government school and low-income students, Class 7 through PhD.
Language: Bengali. Free. Fully online.
Nothing below Class 7 — no serious study for children 12 or younger.


THREE PILLARS (top-level navigation)
--------------------------------------

1. মনের শক্তি       — Mental Resilience
2. ক্যারিয়ার কৌশল — Career Strategy  (absorbs competitive exams + industry/worklife)
3. ভিত্তি (৭-১০)   — Foundations, Class 7-10  [coming soon]

Students naturally move left to right as they age.
Pillars 1 and 2 serve all students throughout their journey.


PILLAR BREAKDOWN
----------------

Pillar 1 — Mental Resilience
  Audience: Everyone
  Topics: exam stress, focus, study habits, failure & recovery, mindset

Pillar 2 — Career Strategy
  Audience: Everyone
  Scope: Everything after "what should I study" — competitive exam paths,
         professional career paths, industry, MBA, entrepreneurship.
  Structure:
    ├── Competitive Exams  (seat-factor exams — limited seats, ranked competition)
    │   Boards (ISC/CBSE) do NOT qualify — marks-only, not seat-limited.
    │     ├── IIT-JEE
    │     │   ├── About the exam (seats, dates, financial aid, college system)
    │     │   ├── Study roadmap (what to study, in what order, by when)
    │     │   ├── Mathematics  → 35 chapters
    │     │   ├── Physics      → chapters
    │     │   └── Chemistry    → chapters
    │     ├── NEET
    │     │   ├── About the exam
    │     │   ├── Biology      → chapters
    │     │   ├── Physics      → SAME content as IIT-JEE Physics, linked not duplicated
    │     │   └── Chemistry    → SAME content as IIT-JEE Chemistry, linked not duplicated
    │     └── (GATE, NET, SET, TET, civil services, recruitment exams — added later)
    │
    └── Worklife  (L5 — industry, professional, post-degree)
          Resume & LinkedIn · Interview prep · Management consulting
          Entrepreneurship · MBA (CAT/GMAT) · Corporate careers

Pillar 3 — Foundations (Class 7-10)
  Audience: Class 7-10 students
  Purpose: school syllabus help + board exam prep (ISC, CBSE, state boards)
  Content: largely separate from Class 11-12 (not a simpler version of it)
  Status: NOT YET READY — placeholder at launch


CONTENT REUSE PRINCIPLE
------------------------
- Each chapter/topic lives in ONE place.
- Multiple exam paths link to the same chapter — content is never duplicated.
- A student entering via IIT-JEE and one entering via NEET reach the same
  Physics chapter page. The page shows which exams it is relevant for.


CHAPTER PAGE STRUCTURE (e.g. Logarithms)
------------------------------------------
  - Concept (theory, intuition, explained in Bengali)
  - Worked examples
  - Practice problems (graded by difficulty)
  - Mock/past exam questions
  - Tags: [IIT-JEE] [NEET] [Class 11] [Class 12]


WHAT NOT TO DO
--------------
- Do not make "Class 11" or "Class 12" a top-level nav item.
  Students think in terms of goals, not classes.
- Do not expose the raw subject/chapter library as a front door.
  That makes it an encyclopedia, not a learning path.
- Do not duplicate content across exam paths.
  Overlap between IIT and NEET subjects is handled by linking, not copying.
- Class 7-10 content is school syllabus help — it is NOT a simpler or
  earlier version of Class 11-12 content. Treat them as separate pillars.

#DEV-PROD-STRATEGY
------------------
Applies to: Bengali_Site and Foundation_Site (both repos)

Branching model: Simple two-branch

  master  →  production (GitHub Pages deploys from here, always live)
  dev     →  feature work accumulates here until ready to ship

Local folder layout (git worktree — one repo, two directories):

  Bengali_Site/         ← master (prod)
  Bengali_Site_dev/     ← dev

  Foundation_Site/4sh-socrates.github.io/   ← master (prod)
  Foundation_Site_dev/                      ← dev

  Both folders share the same .git — no duplicate clones.
  Changes committed in one are immediately visible in the other via git.

Hotfix workflow (small daily nits):
  1. Work in Bengali_Site/ (already on master)
  2. Make the fix, commit, push origin master
  3. Sync to dev:  cd Bengali_Site_dev && git merge master

Feature workflow (larger dev work):
  1. Work in Bengali_Site_dev/ (on dev branch)
  2. Commit and push origin dev as you go
  3. When ready to ship: cd Bengali_Site && git merge dev && git push origin master

Rules:
  - master is always deployable — never commit broken code directly to master
  - dev never auto-deploys — safe to leave work-in-progress there
  - Do not use Gitflow (release/*, develop, etc.) — overkill for this scale
  - Never run "git checkout" inside a worktree to switch branches —
    each worktree is locked to its branch; use the other folder instead


#CONTENT-DEPTH-SYSTEM
----------------------

Principle:
  Every chapter starts at Level 0 and goes deeper.
  Gates are inserted in the chapter content at the point where each exam's
  syllabus ends. A student following an exam path is told exactly where to stop.

Entry point rule:
  A student NEVER browses chapters cold. They always enter via an exam page
  (/iit-jee, /gate, /net etc.), learn about the exam fully, then follow
  chapter links. The exam page controls which chapters appear and to what depth.

The 6 depth levels:

  Level  Tag  Nickname    Exam / Content examples          Descriptor
  -----  ---  ----------  -------------------------------  ----------------------
  0      L0   Noob        ISC, CBSE, State Boards          Foundation / Board
  1      L1   Basics      IIT-JEE, NEET, engg entrances    Competitive / Entrance
  2      L2   Competence  GATE, degree-level exams         Undergraduate
  3      L3   Mastery     NET, SET, TET                    Postgraduate / Masters
  4      L4   Zen         PhD, JRF                         Research
  5      L5   Worklife    Resume, interviews, CAT/GMAT,    Industry / Professional
                          consulting, entrepreneurship,
                          MBA, corporate careers

  Note: L5 is orthogonal to the academic depth axis (L0-L4). It does not
  deepen a subject — it applies knowledge to real-world career contexts.
  L5 content lives under the Worklife sub-section of Career Strategy.

Seat factor rule:
  Only exams with limited seats and ranked competition belong in Pillars 1-4.
  Boards (ISC/CBSE) are L0 prerequisite material only — they appear as
  "Foundations" content, not as competitive exam paths.

How a chapter gate looks (example: Integration):

  [L0 content — basic antiderivatives, standard forms]

  ══ L1 · BASICS ══════════════════════════════════════
     IIT-JEE · NEET · Engineering Entrances
     If you are targeting any of these, stop here.
  ═════════════════════════════════════════════════════

  [L1→L2 content — improper integrals, Beta/Gamma functions]

  ══ L2 · COMPETENCE ══════════════════════════════════
     GATE
     If you are targeting GATE, stop here.
  ═════════════════════════════════════════════════════

  [L2→L3 content — Riemann theory, convergence theorems]

  ══ L3 · MASTERY ═════════════════════════════════════
     NET · SET · TET
     If you are targeting NET/SET/TET, stop here.
  ═════════════════════════════════════════════════════

  [L3→L4 content — Lebesgue integration, measure theory]

Topics with a hard ceiling (example: Linear Programming):
  Content ends at L1. A note reads:
  "This topic does not extend further in standard competitive exam syllabi."

How an exam page uses the level system:
  /iit-jee lists every chapter it needs, annotated with the target level:
    Integration          L0 → L1
    Linear Programming   L0 → L1  (topic ceiling)
    Sets                 L0 → L1
  The student knows exactly how deep to go in each chapter.

Metadata stored per chapter-level pair:
  - level number (0-4)
  - level nickname (Noob / Basics / Competence / Mastery / Zen)
  - exam tags (list of exam names that stop at this level)
  - academic descriptor (Foundation / Competitive / Undergraduate / PG / Research)
  - has_ceiling (true/false — does the topic end at this level?)

Note: Bengali/regional language labels for levels to be decided later.
      Structure is defined in English first.
